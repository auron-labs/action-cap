# Development

How to set up, develop, test, build, and package the ActionCap extension and CLI.

## Repository structure

The repository contains two independent packages:

| Package | Location | Package manager | Purpose |
|---|---|---|---|
| Extension | Repository root | npm | Browser extension |
| CLI | `packages/action-cap-cli/` | Bun | CLI for analyzing exported archives |

The CLI is **not** an npm workspace — it is a standalone package with its own `bun.lock`. Install and run commands from within each package directory.

## Extension development

### Prerequisites

- Node.js with npm
- Chrome, Edge, or Firefox for testing

### Install dependencies

```bash
npm install
```

### Start dev server

```bash
npm run dev
```

This starts the Vite dev server with HMR for extension pages. For extension testing, you will typically also load the built `dist/` folder into the browser after running `npm run build`.

### Build

```bash
npm run build
```

Runs `tsc --noEmit && vite build`. Output goes to `dist/`. Load `dist/` as an unpacked extension in Chrome or Edge.

### Build for Firefox

```bash
npm run build:firefox
```

Sets `ACTIONCAP_BROWSER=firefox` and builds. Load `dist/` as a temporary add-on in Firefox via `about:debugging`.

### Type check

```bash
npm run typecheck
```

Runs `tsc --noEmit`. Run this after any TypeScript changes.

### Testing

The extension has no `npm test` script. The only test file uses Node's built-in test runner:

```bash
node --test src/results/payload-format.test.ts
```

Test files live next to the source they test (e.g., `src/foo/bar.test.ts`). The `tsconfig.json` excludes `src/**/*.test.ts` from the build, so tests do not affect the output.

### No linter or formatter

No ESLint, Prettier, or other linter is configured. Match the existing style:

- 2-space indentation
- Single quotes
- Named imports/exports
- PascalCase for React component files

## Extension packaging

### Package for Edge store

```bash
npm run edge:package
```

Runs `npm run build && node scripts/package-edge.mjs`. Produces:

- `artifacts/edge/actioncap-edge-v<version>.zip`
- `artifacts/edge/actioncap-edge-v<version>.zip.sha256`

Requires the system `zip` CLI to be installed.

### Package for Firefox store

```bash
npm run firefox:package
```

Runs `npm run build:firefox && node scripts/package-firefox.mjs`. Produces:

- `artifacts/firefox/actioncap-firefox-v<version>.zip`
- `artifacts/firefox/actioncap-firefox-v<version>.zip.sha256`

Requires the system `zip` CLI to be installed.

### Generate Edge store assets

```bash
python3 scripts/generate-edge-assets.py
```

Generates promotional images for the Edge store using Pillow. Requires Python 3 with Pillow installed. This is not part of the npm build.

## CLI development

### Prerequisites

- [Bun](https://bun.sh) runtime

### Install dependencies

```bash
cd packages/action-cap-cli
bun install
```

### Run from source

```bash
bun run src/cli.ts --help
bun run src/cli.ts summary recording.bxdac
```

### Build

```bash
bun run build
```

Runs `tsc`. Output goes to `dist/`. The compiled binary is `dist/cli.js`.

### Type check

```bash
bun run typecheck
```

### Testing

```bash
bun test
```

Tests use `bun:test`. The test file is `test/views.test.ts` with a fixture at `test/fixtures/sample.bxdac`.

## Change-aware validation

After making changes, run the relevant checks:

| Change | Command |
|---|---|
| Any TypeScript change (extension) | `npm run typecheck` |
| Any TypeScript change (CLI) | `cd packages/action-cap-cli && bun run typecheck` |
| Changes to `src/results/payload-format.ts` | `node --test src/results/payload-format.test.ts` |
| Before any release-impacting change | `npm run build` |
| Manifest, icons, or packaging changes | `npm run edge:package` and `npm run firefox:package` |
| CLI changes | `cd packages/action-cap-cli && bun test` |

## Generated files

Do not edit these directly — they are generated by build scripts and gitignored:

| Path | Generated by |
|---|---|
| `dist/` | `npm run build` / `npm run build:firefox` |
| `artifacts/edge/` | `npm run edge:package` |
| `artifacts/firefox/` | `npm run firefox:package` |
| `store-assets/` | `scripts/generate-edge-assets.py` (not part of npm build) |

## Dependencies

### Extension

Use `npm`. Do not switch to pnpm or yarn. Add runtime dependencies with `npm install <pkg>` and dev dependencies with `npm install -D <pkg>`. Bundle size matters because the extension ships its own bundle.

### CLI

Use `bun install <pkg>` for dependencies. The CLI's dependencies:

| Package | Purpose |
|---|---|
| `citty` | CLI command framework |
| `@toon-format/toon` | TOON output format serialization |
| `cli-table3` | Table output format |
| `consola` | Logging |
| `pathe` | Path utilities |
| `zod` | Archive schema validation |

## Localization

When adding or changing UI strings:

1. Add keys to both `en-US` and `zh-CN` message objects in `src/common/i18n.ts`.
2. For manifest strings, update both `public/_locales/en/messages.json` and `public/_locales/zh_CN/messages.json`.

## Security guidelines

- Never weaken the sanitizer, masking, or permission logic in `src/common/sanitizer.ts`.
- Never commit real credentials, `.env` files, or exported session files.
- The extension handles extremely sensitive data (passwords, tokens, cookies, request/response bodies, full DOM snapshots).
